Detailed Description

Read and write JSON.

Features:

  • Robust - does not crash or assert() on invalid data.
  • Fast memory allocation - it uses single pooled area for all objects thus alloc/free are fast.
  • Strict UTF8 validation.
  • Full int64_t and double passthrough, except NaN and +-Infinity.
  • Proper number I/O even in weird locales.

Optional features for JSON config files, off by default:

  • Allow C comments.
  • Allow extra comma in dict/list.

enum JsonParseOptions

Options for JSON parser.

Enumerator:
JSON_STRICT 

Default - do strict parsing.

No comments, no extra comma.

JSON_PARSE_RELAXED 

Allow comments, allow extra comma.

Useful for JSON in config files.

JSON_PARSE_IGNORE_ENCODING 

Do not validate UTF-8.

The default behavior is to validate UTF-8.
